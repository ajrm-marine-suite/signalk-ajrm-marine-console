AJRM Marine Help

Onboard reference for the AJRM Marine sailing workflows and safety logic.

Alpha Release disclaimer: This software is Alpha Release and has not been tested in live environments and must not be relied upon for navigation or safety. The Authors do not accept any responsibility for loss or damage as a result of using this software.

AJRM Marine is designed to keep the important traffic information visible and audible without creating unnecessary alarm fatigue. This is the onboard reference for the main sailing workflows and safety logic.

Overview

AJRM Marine Display keeps the chart, target list, alert panel, profile controls, settings, and Help interface focused on sailing use. AJRM Marine Traffic supplies target safety state, CPA/TCPA and profile decisions; AJRM Marine Notifications supplies alert lifecycle and priority.

The map remains usable if GPS is unavailable. In that state the Display keeps chart browsing available and visibly reports the Traffic Core or Signal K data-health state.

Alerts

AJRM Marine Display uses AJRM Marine Traffic and Notifications server-generated alert events for chart colouring, the bottom alert panel, optional popups, Signal K notifications, and sound outputs. The browser, audio plugin, and companion displays do not decide alert severity themselves; they display and speak the event that the provider has already classified.

  • Normal: no attention required.
  • Alert / Watch: a target worth watching, normally visual-only unless sound has been enabled for that state.
  • Traffic advisory: relevant traffic inside the active profile's advisory CPA/TCPA limits. It is the lower audible traffic state.
  • Collision alarm: traffic inside the active profile's danger CPA/TCPA limits, or another high-priority collision condition. This needs more urgent attention.
  • Emergency: distress AIS traffic, such as AIS SART, MOB, or EPIRB targets. Emergency does not depend on CPA/TCPA and is sorted above ordinary traffic alarms.

Repeat timing is controlled by the active profile's repeat sensitivity and the configured repeat intervals. A target that escalates to a higher state can be announced immediately, even if the previous state was recently spoken. The default repeat intervals are 3 minutes for Alert/Watch and Traffic advisory, 1 minute for Collision alarm, and 30 seconds for Emergency before profile repeat sensitivity is applied.

How encounter announcements are built

AJRM Marine builds spoken encounter messages server-side and sends the same wording to the chart, alert panel, Signal K notifications, browser speech, AJRM Marine Audio, and companion displays. The message is intended to explain why the target needs attention, not to replace the skipper's judgement.

  • Severity first: Traffic advisory means the target is inside the active profile's advisory CPA/TCPA limits. Collision alarm means the target is inside the danger limits, or another high-priority collision condition is present.
  • Speed thresholds: CPA/TCPA alert speed checks use the faster of own-vessel SOG and target SOG, so a stationary vessel outside a harbour can still become a collision alarm when own vessel is closing on it.
  • Where to look: clock position is based on the target's current relative bearing from own vessel heading or course.
  • What kind of vessel: AJRM Marine uses AIS name, class, length, and speed where available. Stationary vessels outside harbours are announced as stationary; stationary vessels inside known harbour regions are ignored for encounter alerts.
  • What is happening: closest-approach geometry is described as a prediction: CPA will be on your port side, on your starboard side, ahead, or astern. On the same general course, AJRM Marine says You are overtaking it only when own vessel is faster and the target is ahead or abeam. It says It is overtaking you only when the target is faster and astern or abeam. Otherwise it says Same general course. Stationary targets use You will pass wording because they cannot pass own vessel.
  • How close and when: CPA and TCPA describe the expected closest approach. With hull-aware CPA enabled, AJRM Marine uses vessel dimensions and AIS antenna offsets where available, and a conservative hull envelope when they are missing.
How the geometry wording is chosen
Action prompt sectors

Based on the target's current bearing relative to own vessel. Used only for moving collision-level targets.

Port-bow head-on: alter starboard Starboard sector: Give Way Port and astern sectors: Stand On Stationary collision target: pass well clear
CPA pass wording

Based on where the target is predicted to be at closest approach, relative to own vessel's course.

Ahead: CPA will be ahead Astern: CPA will be astern Starboard: CPA will be on your starboard side Port: CPA will be on your port side
When alerts are suppressed

Suppression is also decided by the server, so all connected displays see the same result.

  • Own vessel: excluded from traffic alerts, even if it appears in the AIS target list. AJRM Marine uses it only as the reference for relative bearing, CPA, TCPA, and profile logic.
  • Base stations and AtoNs: AIS base stations and aids to navigation are classified as non-collision targets and do not create CPA encounter alerts.
  • Out of range: targets outside the configured maximum target range are ignored for active encounter alerts. If a previously active target becomes range-ignored, AJRM Marine publishes a clear event so alert panels and companion displays can remove it.
  • Stationary in harbour: stationary vessels inside known harbour regions are ignored for encounter alerts, even when own vessel is outside the harbour. This prevents anchored, moored, or berthed harbour traffic from being announced as a live collision risk while passing the harbour entrance.
  • Stationary outside harbour: stationary vessels outside known harbour regions are still eligible for alerts. Their announcements include stationary and say You will pass it to port, You will pass it to starboard, You will pass ahead of it, or You will pass astern of it.
  • Silenced or muted: silencing a target or muting sound suppresses speech, not awareness. The target remains visible on the chart and in lists.
Collision avoidance prompt

Action prompt wording is added only to collision-level alerts where CPA geometry indicates collision or close quarters. If the projected geometry predicts a defined pass to port, starboard, ahead, or astern, AJRM Marine reports that CPA geometry and CPA/TCPA without advising a manoeuvre. Traffic advisories do not get Stand On, Give Way, alter-starboard, or pass-well-clear prompts.

  • Alter course early to pass well clear is used for stationary collision targets. AJRM Marine does not apply head-on, give-way, or stand-on wording to stationary targets because they are not crossing under power.
  • Head-on: alter starboard, pass port-to-port is used for moving targets only when the target is dead ahead or on the port bow, from 345 degrees through 360 degrees relative.
  • Give Way is used for collision-level targets in the starboard sector, from just over dead ahead through 112.5 degrees relative.
  • Stand On is used for collision-level targets in the astern or red/port sector, from more than 112.5 degrees through less than 345 degrees relative.

The alter-starboard wording is deliberately narrow. AJRM Marine does not use it for a target already on the starboard bow, because turning further starboard could increase risk if the vessel is expected to pass safely to starboard. Always use the full situation, including visibility, sea room, other traffic, and vessel constraints, before making a manoeuvre.

Lost GPS is treated as a system alert. It repeats every three minutes while GPS remains unavailable unless paused, and the pause is cleared automatically when GPS is received again.

Silencing

Silence all stops current active traffic alerts from speaking without hiding them from the chart. Individual targets can also be silenced from the target list. Silenced vessels remain visible and are marked on the chart and in the list, so the skipper can still see what has been quietened.

Silencing is server-authoritative, so all connected browser clients and companion displays read the same state after refreshing their server data.

Profiles

Profiles hold the CPA/TCPA limits, vessel-size adjustments, and speed thresholds used by AJRM Marine Traffic. Display can select the active Traffic Core profile when commands are available. Threshold editing belongs to the Traffic Core administration web app.

Automatic profile selection is provided by Traffic Core when enabled. Manual profile selection remains available from Display.

Charts

Auto Charts chooses a local Signal K chart provider layer for the current map position and zoom. Basemaps are separate background layers; OpenSeaMap is a transparent seamark overlay and should sit above charts so wind farms, buoys, and other seamarks remain visible.

Manual panning pauses own-vessel following. Press the centre vessel button to recentre on your vessel and resume automatic following.

HTTPS and iPhone Trust

AJRM Marine, Companion, AJRM Marine Harbour Editor, Vessel Simulator, and the Signal K Admin UI are all served by the same Signal K server. Certificate warnings must therefore be fixed at the Signal K server hostname, not inside each webapp.

  1. Give the Raspberry Pi a stable boat-network name, for example nemo3.local, and preferably a fixed LAN IP address.
  2. Install mkcert on the Pi or another trusted computer and create a local certificate authority.
  3. Generate a certificate for every name used to open Signal K, for example nemo3.local, nemo3, and the Pi IP address.
  4. Copy the generated certificate to ~/.signalk/ssl-cert.pem and the private key to ~/.signalk/ssl-key.pem on the Pi. Both files must be readable only by the Signal K user.
  5. Ensure the systemd service starts Signal K with SSLPORT=3443, then restart Signal K.
  6. Install the mkcert root CA on the iPhone. Open the CA file on the iPhone, install the profile in Settings, then enable full trust in Settings > General > About > Certificate Trust Settings.

After that, open Signal K using the same hostname that is in the certificate, for example https://nemo3.local:3443/signalk.

Debugging a Problem

After updating AJRM Marine, the browser can keep an old JavaScript or CSS file in its cache. If the server has been updated but the chart, settings, badge, or help text still looks unchanged, do a hard refresh before assuming the plugin is still wrong.

  • Chrome, Edge, or Firefox on Windows/Linux: press Ctrl+F5 or Ctrl+Shift+R.
  • Chrome, Edge, Firefox, or Safari on Mac: press Cmd+Shift+R. In Safari, Option+Cmd+R also reloads the page from the network.
  • Safari on iPhone or iPad: tap reload first. If the old page remains, clear the site data in Settings > Safari > Advanced > Website Data, search for the Signal K hostname such as nemo3.local, delete that entry, then reopen AJRM Marine.
  • iPad with a hardware keyboard: Cmd+Shift+R can be used in many browsers. If it still shows old behaviour, clear Safari website data for the Signal K hostname.
  • Installed Home Screen webapp: close the webapp from the app switcher after clearing site data, then open it again from the Home Screen.

If a hard refresh does not change the behaviour, check the Signal K plugin version, then capture a fresh AJRM Marine snapshot so the browser state and server state can be compared.

Running Tests

From the repository root, run the full lint, unit-test, and plugin syntax-check suite with: 

npm test

When changing the webapp, rebuild the Signal K-served production assets before release with: 

npm run build:web:recovered
